\section{Problem}
\SectionSlide{Fixed learning rates ignore changing gradient-estimation quality.}

\begin{frame}{Starting Point: Batch Size Controls Gradient Noise}
\small

In stochastic optimization, we do not update with the true gradient \(g_t\).

\vspace{0.5em}

We estimate it from a finite batch:
\[
  \hat g_t = \frac{1}{n}\sum_{i=1}^n X_i .
\]

\vspace{0.5em}

The estimation error decreases with batch size:
\[
  \E\!\left[\|\hat g_t-g_t\|^2\right]
  =
  \frac{\sigma_t^2}{n}.
\]

\vspace{0.8em}

So \(n\) controls how noisy the update direction is.

\vspace{0.5em}

A noisier direction requires a smaller learning rate; a cleaner direction can support a larger one.

\end{frame}

\begin{frame}{What Changes in On-Policy RL?}
\small

\textbf{Supervised learning: data are external}
\[
  (x_i,y_i) \sim \mathcal D,
  \qquad
  \mathcal D \text{ is fixed.}
\]

{\color{Muted}A batch of size \(n\) is sampled from the same underlying data distribution throughout training.}

\vspace{0.7em}

\textbf{On-policy RL: data are self-generated}
\[
  \tau_i \sim \pi_{\theta_t},
  \qquad
  \theta_{t+1} = \theta_t + \alpha \hat g_t .
\]

{\color{Muted}Here \(\tau_i\) is a rollout: one response/trajectory generated by the current model.}

\vspace{0.6em}

After each update, the model that generates the next batch changes:
\[
  \pi_{\theta_t} \longrightarrow \pi_{\theta_{t+1}} .
\]

\vspace{0.6em}

As training succeeds, the policy usually becomes more concentrated:
high-probability responses become more likely, and alternatives are sampled less often.

\end{frame}

\begin{frame}{What Does Policy Concentration Do to the Batch?}
\small

The nominal batch size is still \(n\).

\vspace{0.8em}

But concentrated policies produce more redundant rollouts:
\[
  n_{\mathrm{eff}}(t)
  \;=\;
  \text{non-redundant learning signals}
  \;\leq\; n .
\]

\vspace{0.8em}

So the noise term changes from
\[
  \frac{\sigma_t^2}{n}
  \quad\longrightarrow\quad
  \frac{\sigma_t^2}{n_{\mathrm{eff}}(t)} .
\]

\vspace{0.8em}

If \(n_{\mathrm{eff}}(t)\) changes, the learning rate that was appropriate for one phase may be wrong for another.

\end{frame}

\begin{frame}{Core Research Question}
\small

From the previous slide, the effective noise level is
\[
  \frac{\sigma_t^2}{n_{\mathrm{eff}}(t)} .
\]

\vspace{0.8em}

But standard training often uses a fixed learning rate:
\[
  \theta_{t+1} = \theta_t + \alpha \hat g_t,
  \qquad
  \alpha \text{ is constant.}
\]

\vspace{0.8em}

This creates a mismatch:
\[
  \text{fixed update scale}
  \quad \text{vs.} \quad
  \text{changing batch information quality}.
\]

\vspace{0.8em}

\begin{center}
\Large
\textbf{How should we adapt the update scale to the current batch's effective learning signal?}
\end{center}

\end{frame}

\section{Experimental Results}
\SectionSlide{Fixed learning rates show phase mismatch in on-policy RL.}

\begin{frame}{Experiment 1: Fixed-LR Sweep}
\small

Question:
\[
  \text{Can one fixed learning rate work well across all training phases?}
\]

\vspace{0.8em}

Setup:
\begin{itemize}
  \item model: DeepSeek-R1-Distill-Qwen-1.5B;
  \item training: synchronous PPO/GRPO, 300 steps;
  \item sweep fixed learning rates: \(1\mathrm{e}{-6}\), \(3\mathrm{e}{-6}\), \(5\mathrm{e}{-6}\), \(1\mathrm{e}{-5}\);
  \item repeated runs: 3 seeds where applicable;
  \item metric: AIME validation trajectory.
\end{itemize}

\vspace{0.6em}

What we look for:
\[
  \text{slow learning, best final performance, and late-stage degradation.}
\]

\end{frame}

\begin{frame}{How to Read the Sweep}
\small

For each fixed LR run, we track:

\vspace{0.5em}

\begin{itemize}
  \item \textbf{AIME peak}: best validation score reached during training;
  \item \textbf{Peak step}: when that best score appears;
  \item \textbf{AIME final}: score at step 300;
  \item \textbf{Peak--final}: how much performance is lost after the peak.
\end{itemize}

\vspace{0.8em}

Interpretation:
\[
  \text{large negative Peak--final}
  \quad\Rightarrow\quad
  \text{the run learned something early but could not preserve it.}
\]

\vspace{0.6em}

This is the empirical signature of phase mismatch.

\end{frame}

\begin{frame}{Existing Diagnostics Do Not Measure Batch Information}
\small

\textbf{KL} measures policy movement:
\[
  \text{How far did the new policy move from the old/reference policy?}
\]

\vspace{0.5em}

\textbf{Grad norm} measures update magnitude:
\[
  \text{How large is the raw gradient vector?}
\]

\vspace{0.5em}

\textbf{But our question is different:}
\[
  \text{How reliable is the current batch as an estimate of the useful update direction?}
\]

\vspace{0.7em}

In the LR sweep, the failed \(1\mathrm{e}{-5}\) run has much worse AIME@300 than \(3\mathrm{e}{-6}\), while final KL remains small and final grad norm is not larger.

\vspace{0.5em}

So we need a batch-reliability signal, not only a policy-motion or update-magnitude signal.

\end{frame}

\section{Derivation}
\SectionSlide{The optimal update scale depends on signal-to-noise.}

\begin{frame}{From Smoothness to Expected Improvement}
\small

For a locally \(L\)-smooth objective and update
\[
  \theta_{t+1}=\theta_t+\alpha\ghat,
\]
we have the one-step lower bound
\[
  J(\theta_{t+1})-J(\theta_t)
  \gtrsim
  \alpha g_t^\top \ghat
  -
  \frac{L}{2}\alpha^2\|\ghat\|^2 .
\]

\vspace{0.6em}

Taking expectation over the batch randomness:
\[
  \E[g_t^\top \ghat] = \|g_t\|^2 .
\]

\vspace{0.6em}

The second-order term depends on the noisy update magnitude:
\[
  \E\|\ghat\|^2
  =
  \|\gtrue\|^2
  +
  \E\|\ghat-\gtrue\|^2 .
\]

\end{frame}

\begin{frame}{Insert Effective Batch Noise}
\small

From the previous section,
\[
  \E\|\ghat-\gtrue\|^2
  \approx
  \frac{\sigma_t^2}{n_{\mathrm{eff}}(t)} .
\]

\vspace{0.6em}

Therefore,
\[
  \E\|\ghat\|^2
  \approx
  \|\gtrue\|^2
  +
  \frac{\sigma_t^2}{n_{\mathrm{eff}}(t)} .
\]

\vspace{0.6em}

Plugging this into the bound:
\[
  \E[\Delta J_t]
  \gtrsim
  \alpha \|\gtrue\|^2
  -
  \frac{L}{2}\alpha^2
  \left(
    \|\gtrue\|^2
    +
    \frac{\sigma_t^2}{n_{\mathrm{eff}}(t)}
  \right).
\]

\end{frame}

\begin{frame}{Optimal LR Contains a Signal Fraction}
\small

The bound is a quadratic function of \(\alpha\):
\[
  f(\alpha)
  =
  \alpha A
  -
  \frac{L}{2}\alpha^2 B,
  \quad
  A=\|\gtrue\|^2,\quad
  B=\|\gtrue\|^2+\sigma_t^2/n_{\mathrm{eff}}(t).
\]

\vspace{0.4em}

Taking derivative and setting it to zero:
\[
  f'(\alpha)=A-LB\alpha=0.
\]

\vspace{0.4em}

Thus
\[
  \alpha_t^*
  =
  \frac{\|\gtrue\|^2}
       {L\left(
          \|\gtrue\|^2
          +
          \sigma_t^2/n_{\mathrm{eff}}(t)
        \right)} .
\]

\vspace{0.5em}

Define the signal fraction
\[
  r_t =
  \frac{\|\gtrue\|^2}
       {\|\gtrue\|^2+\sigma_t^2/n_{\mathrm{eff}}(t)}
  \in (0,1].
\]

\vspace{0.8em}

Then
\[
  \alpha_t^* = \frac{r_t}{L}.
\]

\end{frame}

\begin{frame}{Interpretation}
\small

\[
  r_t =
  \frac{\text{gradient signal}}
       {\text{gradient signal}+\text{estimation noise}}
\]

\vspace{0.8em}

If the current batch contains many non-redundant learning signals, \(n_{\mathrm{eff}}(t)\) is larger and \(r_t\) is larger.

\vspace{0.6em}

If rollouts become redundant or weakly informative, \(n_{\mathrm{eff}}(t)\) is smaller and \(r_t\) is smaller.

\end{frame}

\section{Method}
\SectionSlide{Estimate signal fraction from within-batch agreement and rescale LR.}

\begin{frame}{Why Split the Batch?}
\small

We want the signal fraction
\[
  r_t =
  \frac{\|\gtrue\|^2}
       {\|\gtrue\|^2+\sigma_t^2/n_{\mathrm{eff}}(t)} .
\]

\vspace{0.8em}

But \(\gtrue\), \(\sigma_t^2\), and \(n_{\mathrm{eff}}(t)\) are not directly observable.

\vspace{0.8em}

So we create two independent estimates of the same gradient:
\[
  \hat g_A = g_t+\epsilon_A,
  \qquad
  \hat g_B = g_t+\epsilon_B .
\]

\vspace{0.8em}

Common direction across the two halves is signal.

Disagreement between the two halves is noise.

\end{frame}

\begin{frame}{Split Agreement Estimates Signal Fraction}
\small

Because the split noises are independent,
\[
  \E[\hat g_A^\top \hat g_B]
  =
  \|g_t\|^2 .
\]

\vspace{0.6em}

Each half's squared norm contains both signal and noise:
\[
  \E\|\hat g_A\|^2
  \approx
  \|g_t\|^2+\text{noise}.
\]

\vspace{0.6em}

Therefore we estimate
\[
  \rhat =
  \frac{\hat g_A^\top \hat g_B}
       {\frac{1}{2}\left(\|\hat g_A\|^2+\|\hat g_B\|^2\right)}
  \approx
  \frac{\text{signal}}{\text{signal}+\text{noise}} .
\]

\vspace{0.6em}

High agreement means reliable batch signal; low agreement means noisy or weak signal.

\end{frame}

\begin{frame}{Algorithm in Practice}
\small

\begin{algorithmic}[1]
\STATE Sample on-policy rollout batch \(D_t \sim \pi_{\theta_t}\)
\STATE Split prompt groups: \(D_t = A \cup B\)
\STATE Compute half-batch gradients \(\hat g_A,\hat g_B\)
\STATE Estimate signal fraction
\[
  \rhat =
  \frac{\hat g_A^\top \hat g_B}
       {\frac{1}{2}\left(\|\hat g_A\|^2+\|\hat g_B\|^2\right)}
\]
\STATE Smooth / clip / guard \(\rhat\)
\STATE Form stabilized control signal \(r_{\mathrm{ctrl},t}\)
\STATE Set learning rate \(\alpha_t = c_{\mathrm{fix}} r_{\mathrm{ctrl},t}\)
\STATE Update policy: \(\theta_{t+1} = \theta_t + \alpha_t \hat g_{\mathrm{upd}}\)
\end{algorithmic}

\vspace{0.5em}

{\color{Muted}The estimator reuses the ordinary rollout batch; no extra rollout stream is required.}

\end{frame}

\begin{frame}{Adaptive LR Rule: Current Experimental Form}
\small

The ideal rule is
\[
  \alpha_t = c_t \rhat .
\]

\vspace{0.6em}

In the current experiments, we fix the scale:
\[
  c = c_{\mathrm{fix}},
  \qquad
  \alpha_t = c_{\mathrm{fix}} \cdot r_{\mathrm{ctrl},t}.
\]

\vspace{0.4em}

The value of \(c_{\mathrm{fix}}\) is selected by a prior sweep.

\vspace{0.6em}

The control signal is the stabilized signal fraction:
\[
  r_{\mathrm{ctrl},t}
  =
  \mathrm{clip}\!\left(
    \mathrm{EMA}(\rhat),
    r_{\min},
    r_{\max}
  \right).
\]

\vspace{0.5em}

\textbf{Purpose of this setting}

\begin{itemize}
  \item remove the uncertainty of learning the global scale \(c_t\);
  \item isolate whether the batch-dependent \(r_t\) schedule helps;
  \item use EMA / clipping / sign guard only to stabilize the \(r_t\) estimate.
\end{itemize}

\end{frame}

\begin{frame}{Claim Boundary}
\small

\textbf{Main conclusion:}
with \(c\) fixed, signal fraction provides a useful update-scale schedule.

\vspace{0.8em}

\textbf{Supported by current 1.5B experiments}

\begin{itemize}
  \item Beats matched constant LR \(\Rightarrow\) mean LR is not enough.
  \item D3 beats constants \(\Rightarrow\) schedule shape matters.
  \item Signal fraction beats D3 \(\Rightarrow\) online state dependence still helps.
\end{itemize}

\vspace{0.9em}

\textbf{Boundary}

These runs validate the \(r_t\)-side schedule, not yet the full adaptive \(c_t\) controller.

\end{frame}

\small

\begin{itemize}

  \item This mostly gives a one-way schedule: LR can shrink when \(r_t\) falls.
  \item Next: fit/update \(c_t\) slowly so the system can also increase the global scale when updates are too conservative.
\end{itemize}

\end{frame}

\begin{frame}{One-Slide Summary}
\small

\begin{enumerate}
  \item In stochastic optimization, LR should depend on gradient-estimator quality.
  \item In on-policy RL, the data distribution changes with \(\pi_{\theta_t}\), so effective batch information is non-stationary.
  \item Fixed LR therefore creates update-scale / batch-information mismatch.
  \item Standard diagnostics such as KL and grad norm do not fully reveal this mismatch.
  \item We estimate current batch reliability by split-batch signal fraction.
  \item Current experiments use \(\alpha_t=c_{\mathrm{fix}}r_{\mathrm{ctrl},t}\) to isolate the value of the batch-dependent signal.
\end{enumerate}

\vspace{0.8em}

\takeaway{
The project is an optimization problem: adapting update scale to non-stationary stochastic gradient quality in self-generated on-policy data.
}

\end{frame}
